MFG Lab · bring your own game

Type a mean-field game. Get the certified equilibrium.

Supply the two things that define the game — the coupling f(m), how a player's cost depends on the crowd's local density, and the terminal cost V(x), where the crowd is trying to go. The page solves the coupled HJB / Fokker–Planck system by fictitious play and returns the equilibrium density, the value function, and the exploitability: the ε-Nash gap that says whether the fixed point it reached is really an equilibrium, or only looks like one. Runs in this page; nothing is uploaded.

What this is. The solver is lab/mfg-byo.js — the same FD-monotone fictitious play validated in the lab's Module 1, generated into this page and gated on byte-identity, so what runs here is what the battery proved. The certificate is exploitability, a pre-update best-response gap: it is honest for any cost you type. A well-posed (monotone) coupling certifies an equilibrium; a pathological one reports a large ε or stalls, and the page says not an equilibrium rather than pretending.

The same solver in Python is planned as mfglab.mfg; today it is JavaScript here and in the battery.
